Dynamic vs. Static Typed Languages

Let's start off by defining the two terms, then jump into advantages and disadvantages of each type, and finally end off with our analysis.

Static Typed Language

When setting a variable in a static typed language, you have to know what type of variable it is when the code runs. This means that when you're for example saying var number = 2, you'll have to do int var number = 2, and explicitly state that the value of that variable is going to be an integer.

Advantages & Drawbacks

The advantage to writing it out explicitly is that Type Errors are caught earlier and more difficult to make.

Dynamic Typed Language

When setting variables in a dynamic language, you don't have to tell the compiler what type of value to expect, it will just know that 2 is an integer and 'test' is a string.

the advantage here is that it's less verbose and quicker to write with dynamic programming languages. The drawback is that you run the risk of runtime Type Erros
