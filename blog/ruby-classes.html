Ruby Classes

Setting up classes with instance variables and methods

Setting up a class:

class PlayingCard
SUITS = %w{ clubs diamonds hearts spades }
RANKS = %w{2345678910JQKA}
class Deck def cards
@cards.to_enum
end
def initialize(n=1) @cards = [] SUITS.cycle(n) do |s|
RANKS.cycle(1) do |r| @cards << "#{r} of #{s}"
end
end
end
end
end

Using instance variables and methods makes sense when those objects are local to your class. If they need to be accessed globally, then it makes sense to use a global variable.

It's useful to organize your program this way because when variables and methods are local to the class, you can have smaller functions that do and return one value, and manipulate them through the class.
